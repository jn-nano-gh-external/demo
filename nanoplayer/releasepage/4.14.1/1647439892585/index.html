Release Notes

This feature release is introducing two new latency control modes especially targeting lowest latency use cases like live auctioning. The latency control mode can be selected via the added config.playback.latencyControlMode configuration parameter. The established latency control mode is available as the 'classic' option and is remaining the default mode. Furthermore the new 'balancedadaptive' and 'fastadaptive' latency control modes have been added, which are capable of adjusting the latency adaptively according to the current stream and network conditions, to achieve lower latency while keeping the playback experience smooth. Please find further information in our feature description in the documentation.
